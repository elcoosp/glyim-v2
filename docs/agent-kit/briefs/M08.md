# Stream M08: Move Integration Tests out of glyim-test/lib.rs

## Mission
Implement Move the large `tests` module (1400+ lines) from `crates/glyim-test/src/lib.rs` into separate test files under `tests/` directory. Keep `lib.rs` only for public API and helper modules. Ensure all integration tests still run via `cargo test -p glyim-test`. for the Glyim compiler.

## Test Location (MANDATORY)

**ALL tests MUST be written as unit tests in `src/tests/` within the crate.**

NEVER use the `tests/` directory at the crate root (integration tests). Unit tests compile faster because they share the crate's private API.

### Setup Pattern

1. In `crates/glyim-test/src/lib.rs`, add:
   ```rust
   #[cfg(test)]
   mod tests;
   ```

2. Create `crates/glyim-test/src/tests/mod.rs` declaring submodules:
   ```rust
   mod module_a;
   mod module_b;
   ```

3. Each test file contains `#[test]` functions with full access to `pub(crate)` items.

4. Add `glyim-test` as a dev-dependency:
   ```toml
   [dev-dependencies]
   glyim-test = { workspace = true }
   ```

5. Read `docs/agent-kit/GLYIM_TEST_INSTRUCTIONS.md` for full API reference.
