# Stream S22: LLVM Backend Unstubbing

## Mission
Implement Remove all `STUB:` warnings in `glyim-codegen-llvm`. Map all `TyKinds` to proper LLVM types (error->i64, floats->f32/f64, aggregates->structs/arrays). Implement `lower_aggregate` for ADTs/closures, `lower_len`, `lower_repeat`. Implement string constant lowering via global string pointer, function constant lowering via function pointer, and const ref lowering via global address. Implement `Deref` on non-pointer, `Field` projection on non-aggregate using `adt_def`, `Index` projection on non-array/slice, `Len` with non-integer count, logical `Not` on float (emit trap), `Deref` on float (emit trap), `Repeat` with non-integer count, `Assert` terminator, and unhandled `PassMode` arguments (`Indirect`, `Cast`, `HomogeneousAggregate`, `Split`, `Ignore`) per ABI rules. for the Glyim compiler.

## Test Location (MANDATORY)

**ALL tests MUST be written as unit tests in `src/tests/` within the crate.**

NEVER use the `tests/` directory at the crate root (integration tests). Unit tests compile faster because they share the crate's private API.

### Setup Pattern

1. In `crates/glyim-codegen-llvm/src/lib.rs`, add:
   ```rust
   #[cfg(test)]
   mod tests;
   ```

2. Create `crates/glyim-codegen-llvm/src/tests/mod.rs` declaring submodules:
   ```rust
   mod module_a;
   mod module_b;
   ```

3. Each test file contains `#[test]` functions with full access to `pub(crate)` items.

4. Add `glyim-test` as a dev-dependency:
   ```toml
   [dev-dependencies]
   glyim-test = { workspace = true }
   ```

5. Read `docs/agent-kit/GLYIM_TEST_INSTRUCTIONS.md` for full API reference.
