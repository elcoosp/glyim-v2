# Stream S21: Bytecode Backend Unstubbing

## Mission
Implement Remove all `STUB:` warnings in `glyim-codegen`. Implement `Assign` with projections by computing target address offset. Implement `Rvalue::Ref` with projection and projected Operands. Support `String`, `Fn`, and `ConstRef` constant kinds via allocation tables. Implement `Assert` terminator with conditional branching to cleanup/panic, and `Drop` terminator with projection via drop glue lookup. for the Glyim compiler.

## Test Location (MANDATORY)

**ALL tests MUST be written as unit tests in `src/tests/` within the crate.**

NEVER use the `tests/` directory at the crate root (integration tests). Unit tests compile faster because they share the crate's private API.

### Setup Pattern

1. In `crates/glyim-codegen/src/lib.rs`, add:
   ```rust
   #[cfg(test)]
   mod tests;
   ```

2. Create `crates/glyim-codegen/src/tests/mod.rs` declaring submodules:
   ```rust
   mod module_a;
   mod module_b;
   ```

3. Each test file contains `#[test]` functions with full access to `pub(crate)` items.

4. Add `glyim-test` as a dev-dependency:
   ```toml
   [dev-dependencies]
   glyim-test = { workspace = true }
   ```

5. Read `docs/agent-kit/GLYIM_TEST_INSTRUCTIONS.md` for full API reference.
