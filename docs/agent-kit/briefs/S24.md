# Stream S24: Compilation Pipeline Unstubbing

## Mission
Implement Remove all `STUB:` warnings in `glyim-pipeline`. Implement generic instantiation by cloning MIR body and replacing `ParamTy`/`ParamConst`/`EarlyBoundRegion` with concrete args from `Substitution`. Change missing MIR body fallback to return a compilation error instead of a dummy body. Implement drop glue generation by walking type fields and emitting `Drop` terminators; add a `DropGlueProvider` trait and implement it. Fix ADT field type resolution in `pipeline_context` to use `TyCtx::adt_def` instead of error types. for the Glyim compiler.

## Test Location (MANDATORY)

**ALL tests MUST be written as unit tests in `src/tests/` within the crate.**

NEVER use the `tests/` directory at the crate root (integration tests). Unit tests compile faster because they share the crate's private API.

### Setup Pattern

1. In `crates/glyim-pipeline/src/lib.rs`, add:
   ```rust
   #[cfg(test)]
   mod tests;
   ```

2. Create `crates/glyim-pipeline/src/tests/mod.rs` declaring submodules:
   ```rust
   mod module_a;
   mod module_b;
   ```

3. Each test file contains `#[test]` functions with full access to `pub(crate)` items.

4. Add `glyim-test` as a dev-dependency:
   ```toml
   [dev-dependencies]
   glyim-test = { workspace = true }
   ```

5. Read `docs/agent-kit/GLYIM_TEST_INSTRUCTIONS.md` for full API reference.
