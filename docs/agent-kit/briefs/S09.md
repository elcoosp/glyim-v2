# Stream S09: HIR Lowering Unstubbing

## Mission
Implement Exhaustively match all `Expr`, `BinOp`, `Pat`, and `TypeRef` variants in the HIR lower module, removing all catch-all match arms that produce `Pat::Err` or `Expr::Err`. Implement field name resolution using struct definition order. Support bodyless FnDef for foreign functions instead of panicking. Change the `lower_crate_for_pipeline` signature to return `(CrateHir, Vec<GlyimDiagnostic>)` instead of just `CrateHir` to capture diagnostics during lowering. for the Glyim compiler.

## Test Location (MANDATORY)

**ALL tests MUST be written as unit tests in `src/tests/` within the crate.**

NEVER use the `tests/` directory at the crate root (integration tests). Unit tests compile faster because they share the crate's private API.

### Setup Pattern

1. In `crates/glyim-hir/src/lib.rs`, add:
   ```rust
   #[cfg(test)]
   mod tests;
   ```

2. Create `crates/glyim-hir/src/tests/mod.rs` declaring submodules:
   ```rust
   mod module_a;
   mod module_b;
   ```

3. Each test file contains `#[test]` functions with full access to `pub(crate)` items.

4. Add `glyim-test` as a dev-dependency:
   ```toml
   [dev-dependencies]
   glyim-test = { workspace = true }
   ```

5. Read `docs/agent-kit/GLYIM_TEST_INSTRUCTIONS.md` for full API reference.
