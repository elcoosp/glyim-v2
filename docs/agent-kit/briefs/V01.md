# Stream SV01: Associated Types & Projections

## Mission
Implement Add associated type declarations to traits, projection types (T::Item), projection equality in unification and type checker. Implement occurs check for projections to prevent cycles. Provide diagnostics for unresolved projections. for the Glyim compiler.

## What You Own
- glyim-type
- glyim-solve
- glyim-typeck

## What You May NOT Modify
Any `pub` interface listed in `CONTRACTS_LOCKED.md` for crates you do NOT own.
Specifically, you MUST NOT change:
- glyim_type::TyKind::Projection
- glyim_type::TyKind::Opaque
- glyim_solve::InferenceTable::unify
- glyim_typeck::typeck_crate

## What You May Do Freely
- Add `pub(crate)` helper functions in crates you own
- Add new private modules in crates you own
- Add new `pub` items in NEW modules you create within crates you own
- Add dev-dependencies for testing

## Test Location (MANDATORY)

**ALL tests MUST be written as unit tests in `src/tests/` within the crate.**

NEVER use the `tests/` directory at the crate root (integration tests). Unit tests compile faster because they share the crate's private API.

### Setup Pattern

1. In `crates/glyim-type/src/lib.rs`, add:
   ```rust
   #[cfg(test)]
   mod tests;
   ```

2. Create `crates/glyim-type/src/tests/mod.rs` declaring submodules:
   ```rust
   mod module_a;
   mod module_b;
   ```

3. Each test file contains `#[test]` functions with full access to `pub(crate)` items.

4. Add `glyim-test` as a dev-dependency:
   ```toml
   [dev-dependencies]
   glyim-test = { workspace = true }
   ```

5. Read `docs/agent-kit/GLYIM_TEST_INSTRUCTIONS.md` for full API reference.

## Test Plan (Write These FIRST)
- V01-T01: Define trait `Iterator { type Item; }` with impl `type Item = u32` → projection `I::Item` resolves to `u32` (run-pass)
- V01-T02: Nested projection: `T::Item::Target` where `T::Item` is another trait with associated type → resolves correctly (run-pass)
- V01-T03: Unify `I::Item` with `u32` where `I` bound to impl → Ok (run-pass)
- V01-T04: Projection in where clause: `fn foo<T>() where T::Item: Clone` → compiles (run-pass)
- V01-T05: Associated type defaults: `type Item = ()` → works (run-pass)
- V01-T06: Cycle detection: `type A = Self::B; type B = Self::A;` → error (compile-fail)
- V01-T07: Unresolved projection → error diagnostic (compile-fail)
- V01-T08: Projection with mismatched trait bound → error (compile-fail)

## Mocking Strategy
No mocking. Extend existing TyCtx and InferenceTable. Use `glyim_test::assert_ty` and `assert_diag_contains`.

## Implementation Priority
